# Security-check — изоляция и доступ

Главная угроза проекта — водитель увидел или изменил чужую задачу (IDOR). Проверка обязательна, формальная, по чек-листу. Результат фиксируй в отчёте этапа.

## 1. Статический проход по коду

Для КАЖДОГО route handler в `src/app/api/**`:

- [ ] Первой строкой — `requireUser` / `requireRole`. Эндпоинтов без аутентификации нет (кроме `/api/auth/*`).
- [ ] Личность берётся только из сессии. Грепни и убедись, что `userId`, `assigneeId`, `role` НЕ читаются из body/query как идентификатор текущего пользователя: `rg "body\.(userId|assigneeId|role)" src/app/api`.
- [ ] Выборки для роли DRIVER содержат `assigneeId: user.id` в where. Грепни все `prisma.task.findMany` и проверь каждый вызов из водительских маршрутов.
- [ ] Доступ к объекту по id проходит через `assertCanView` / `assertCanTransition` из `src/domain/authz.ts`, а не ручные if-ы.
- [ ] Чужой объект возвращает 404 (не 403 и не пустой 200).
- [ ] Файлы (`/api/attachments/:id`) раздаются с проверкой прав; в `public/` и next/static нет пользовательских загрузок.
- [ ] Смена статуса проверяет допустимость перехода по матрице ИЗ домена (не копия логики в handler).

## 2. Динамические тесты (обязательные, Playwright/Vitest)

Сид создаёт: диспетчера, водителя A, водителя B, задачу для A и задачу для B. Тесты:

- [ ] B запрашивает `GET /api/my/tasks` — задач A в ответе нет.
- [ ] B запрашивает `GET /api/tasks/{id задачи A}` — 404.
- [ ] B шлёт `POST /api/tasks/{id задачи A}/transition` — 404/ошибка, статус задачи A не изменился, событие не создано.
- [ ] B запрашивает фото задачи A по прямому id — 404.
- [ ] Водитель пытается `POST /api/tasks` (создание) — отказ: создаёт только диспетчер/админ.
- [ ] Переход в обход матрицы (ACCEPTED → DONE) — отказ `FORBIDDEN_TRANSITION` для любой роли.
- [ ] DONE для типа `ON_SITE` без отметки оплаты (получено / не получено с причиной) — отказ `PAYMENT_REQUIRED`. (Фото при DONE — по желанию, гейта нет: правило №4 CLAUDE.md, этап 11.)
- [ ] Неаутентифицированный запрос к любому /api/* (кроме auth) — 401.

## 3. Общая гигиена

- [ ] `rg -i "(password|secret|api_key|token)\s*[:=]\s*['\"]" src/` — хардкоженных секретов нет; `.env` в `.gitignore`.
- [ ] Пароли — argon2id; rate limit на вход действует (проверь: 10 неверных попыток подряд → блокировка/задержка).
- [ ] Prisma — параметризация по умолчанию; сырые `\$queryRaw` отсутствуют или параметризованы.
- [ ] Загрузка файлов: проверка mime/размера (фото ≤ 15 МБ), имя файла генерируется сервером (uuid), путь не берётся от клиента.
- [ ] Заголовки: httpOnly+secure+sameSite на сессионной куке.
- [ ] В пушах нет чувствительных данных сверх необходимого (номер задачи, краткий заголовок).

В конце выведи таблицу: пункт → ок/проблема → что исправлено. Если хоть один пункт «проблема» и не исправлен — этап сдавать нельзя.
