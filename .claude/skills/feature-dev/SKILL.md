# Реализация фичи — обязательный цикл

## 1. Перед кодом

1. Прочитай релевантные разделы `docs/PRD.md` (что должно получиться) и `docs/ARCHITECTURE.md` (как это устроено). Найди этап в `docs/ROADMAP.md` — работай в его границах, не затаскивай соседние этапы.
2. Если требование неоднозначно — сформулируй Артёму вопрос ПРО ПРОДУКТ (не про технику) с вариантами и своей рекомендацией. Технические решения принимай сам в рамках ARCHITECTURE.
3. Набросай короткий план: какие модели/домен/эндпоинты/экраны затрагиваются. Если затрагивается статусная матрица или права — остановись и согласуй с Артёмом.

## 2. Порядок реализации

Всегда от данных к интерфейсу:

1. Prisma-схема → миграция (`pnpm prisma migrate dev --name <что_изменилось>`).
2. Домен (`src/domain/`): логика + authz-проверки. Бизнес-правила живут здесь, не в handlers и не в компонентах.
3. Unit-тесты домена (Vitest) — особенно на запрещённые случаи (недопустимый переход, чужая задача, DONE без фото).
4. Route handler — тонкий: parse → вызов домена → ответ `{ data } | { error }`.
5. UI по skill **ui-guidelines**. Для водительских экранов — проверка в мобильном вьюпорте.
6. e2e (Playwright) для нового пользовательского сценария, если он критический (создание задачи, смена статуса, фото, вход).

## 3. После кода

1. `pnpm lint && pnpm typecheck && pnpm test` — всё зелёное.
2. Если менялись API/БД/права → прогони skill **security-check** полностью.
3. Самопроверка от лица пользователя: пройди сценарий руками как Милена и как водитель (два разных логина из сида).
4. Обнови документы, если реальность разошлась со спекой (или предложи Артёму правку спеки).
5. Напиши Артёму итог: что сделано (по-человечески, без жаргона), как посмотреть (url/действия), какие продуктовые вопросы открылись.

## Антипаттерны (не делать)

- Логика в компонентах/handlers вместо `src/domain/`.
- «Времянка без теста, потом починю» для прав доступа и статусов — эти две зоны без тестов не сдаются.
- Новые зависимости/сервисы без обоснования (CLAUDE.md, правило 6).
- Тихое изменение поведения, описанного в PRD.
