# UI-гайдлайн vanmark-drive

Цель: быстрый, чистый, профессиональный интерфейс. Ничего «креативного» в ущерб скорости считывания: Милена живёт в этом экране весь день, водитель смотрит на телефон на улице/за рулём (на светофоре).

## Базовые правила

- Библиотека: shadcn/ui + Tailwind. Не изобретать свои контролы, где есть готовые.
- Шрифт: Inter (или системный стек). Размеры: водитель — базовый 16–18px, кнопки статусов крупные; диспетчер — базовый 14–15px, плотная информация.
- Тач-цели у водителя ≥ 48×48px, главная кнопка статуса — на всю ширину экрана, в зоне большого пальца (нижняя треть).
- Светлая тема в MVP. Без анимаций дольше 200 мс.
- Весь интерфейс на русском. Тексты короткие, без канцелярита: «Выехал», а не «Подтвердить начало движения».

## Цвета статусов (единые ВЕЗДЕ: доска, карточки, бейджи, push)

| Статус | Цвет | Tailwind |
|---|---|---|
| Новая | серый | slate-400 |
| Назначена | фиолетовый | violet-500 |
| Принята | индиго | indigo-500 |
| В пути | синий | blue-500 |
| На месте | оранжевый | orange-500 |
| Выполнена | зелёный | green-600 |
| Ждёт | жёлтый | amber-500 |
| Перенесена | серо-голубой | sky-400 |
| Отменена | красный | red-500 |

Статус всегда показывается цветом + текстом (не только цветом). Срочная задача — красная точка/рамка слева на карточке.

## Карточка задачи (список)

Порядок считывания за 2 секунды: **№ и тип (иконка)** → **время/окно** → **суть (title)** → **адрес** → **статус-бейдж**. У водителя дополнительно крупно: индикатор пропуска («Пропуск заказан» зелёным / «Нужен пропуск!» жёлтым), кнопки «Навигатор» и «Позвонить» прямо с карточки.

## Экраны водителя — особые правила

- Один основной сценарий на экран. Никаких табов-в-табах.
- Кнопка следующего статуса — одна, большая, с текстом действия («Выехал →»). Прошедшие статусы — лента сверху.
- Оптимистичный UI: нажал — статус сразу сменился визуально; при ошибке сети — заметный, но спокойный индикатор «не отправлено, повторяю…» и автоповтор.
- Фото: кнопка открывает камеру сразу (capture), превью добавленных, удаление до отправки.
- Пустые состояния с человеческим текстом: «На сегодня задач нет 🎉» — допустимое единственное место с эмодзи.

## Экран «Сегодня» диспетчера

- Колонки по водителям + «Не назначено» слева. Карточки перетаскиваются (dnd) для назначения/переназначения; переключатель «по статусам».
- Сверху счётчики: всего / в работе / выполнено / требуют внимания (просрочки + пропуска на завтра).
- Обновление данных не должно дёргать раскладку (никаких прыжков при поллинге — стабильные ключи, скелетоны только при первой загрузке).
- Создание задачи — кнопка «+ Задача» в шапке, форма в модалке/drawer, autofocus на первом поле, Enter-флоу, «создать и ещё одну».

## Перед сдачей любого UI

- [ ] Прогон в 360×740 (Android) для водительских экранов и 1366×768+ для доски.
- [ ] Все состояния: загрузка / пусто / ошибка / длинные строки (адрес в 2 строки, название ООО длинное).
- [ ] Цвета статусов из таблицы, не «похожие».
- [ ] Нет английских текстов и плейсхолдеров lorem.
